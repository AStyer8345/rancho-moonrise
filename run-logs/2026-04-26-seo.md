## What Was Built

### `safari-tents-near-austin.html` — FAQPage schema + visible FAQ section

**Audit finding.** Coverage check across the SEO cluster showed safari-tents was the only landing page with `aeo-block (2) | Speakable (2) | BreadcrumbList (2)` but **no FAQPage**. Other pages (things-to-do-manor-tx, glamping-vs-camping, pool-day-pass-austin, things-to-do-near-austin-with-kids) all carried FAQPage. This is a meaningful AEO gap on a high-intent page (a city-anchored accommodations landing) — and one that compounds when DNS cuts over and crawlers see the page for the first time.

**Three edits:**

1. **JSON-LD FAQPage schema block** — inserted between BreadcrumbList and SpeakableSpecification. 4 questions, answers grounded in existing on-page copy:
   - "What is a safari tent?" — definition + included amenities
   - "How is a safari tent different from a regular camping tent?" — comparison framing
   - "Are safari tents heated and air-conditioned?" — specific amenity (high commercial intent)
   - "Where can I find a safari tent near Austin, Texas?" — local intent, includes Manor TX 78653 in schema body (per Ashley's rule: Manor allowed in tags/schema/alt text, not body copy)

2. **Visible FAQ section** — inserted between the Location section and the Social-Proof CTA, using the standard `.faq-item` / `.faq-question` / `.faq-answer` markup. `main.js:80` already binds the accordion toggle to `.faq-item`, so no JS change needed. Visible Q&A is Google's prerequisite for FAQ rich results — schema without on-page content risks a manual action.

3. **Speakable cssSelector** — expanded from `[".aeo-block", ".page-header h1", ".page-header p"]` to also include `.faq-question`, so AI-engine speakable extraction picks up the new Q&A.

**Verification:**
- `python3` JSON parse: 4 of 4 JSON-LD blocks parse clean (LodgingBusiness ✓, BreadcrumbList ✓, FAQPage ✓, WebPage/Speakable ✓).
- Schema counts after edit: `aeo-block: 2 | FAQPage: 1 | Speakable: 2 | BreadcrumbList: 2 | LodgingBusiness: 2`.
- No banned voice-guide terms introduced. "Manor" appears only in the FAQPage schema answer (rule-compliant).
- DNS still on Flywheel — change lands on Vercel preview (`rancho-moonrise.vercel.app`) for now. Will surface to crawlers post-cutover.
