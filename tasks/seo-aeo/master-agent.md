## AEO Principles (apply to ALL content)
1. **Answer-first content** — Every H2 section leads with a direct, extractable answer in the first sentence. No preamble.
2. **Entity authority** — Rancho Moonrise must be defined as an entity:
   - LodgingBusiness + EventVenue schema with geo, amenities, reviews
   - Consistent NAP across every page footer
   - "Rancho Moonrise is a 34-acre glamping and events ranch in Manor, TX, 20 minutes from downtown Austin."
3. **Conversational query targeting** — Optimize for AI queries:
   - "Best glamping near Austin TX" → glamping page + homepage
   - "Unique wedding venues Austin Texas" → weddings page
   - "Pool day pass Austin" → dedicated pool page or section
   - "Corporate retreat venue near Austin" → corporate page
   - "Things to do in Manor TX" → local guide page
4. **Topical authority clusters** — Deep interlinked coverage:
   - Glamping cluster: glamping page → packing guide → glamping vs camping → Austin outdoor activities
   - Wedding cluster: weddings page → ranch wedding tips → wedding venues near Austin → bachelorette
   - Events cluster: events page → corporate retreats → host your event → things to do Manor TX
5. **Speakable structure** — Key pages have a clear 1-2 sentence summary in the first 150 words that an AI could quote verbatim.
6. **Blog posts as individual pages** — Every blog post gets its own URL, its own Article schema, and links back to the parent cluster page. Blog index page gets CollectionPage schema linking to all posts.
